# Jeeves — AI Governance Gateway (Demo)

> **This is a fictional demo.** "Meridian Health" is not a real company and
> has no affiliation with any real healthcare payer. Every initiative,
> actor, approval, control, and telemetry series in this repository is
> synthetic, generated by a deterministic seed script. No real PHI, PII, or
> production data of any kind is used anywhere in this codebase. Live LLM
> calls, when enabled, only ever see synthetic seed data — never real member
> information.

Jeeves is a demo of an **AI-governance gateway** for a healthcare payer: a
single place to submit an AI initiative, triage its risk deterministically,
route it through the right governance domains, get agent-drafted reviews
that a human signs, issue a named accountable approval, generate versioned
controls for the deployment, and keep monitoring it afterward — with a
structured, evidence-linked audit trail throughout.

The core idea it demonstrates: **agents draft, humans decide.** Agents never
hold approval authority anywhere in this system.

## Architecture sketch

```
app/ (Next.js App Router — UI pages + app/api/** route handlers)
  │
  ├─▶ lib/services   transactional use cases, gated by the route-guard
  │                  pipeline (session → rate-limit → budget)
  │
  ├─▶ lib/domain, triage, approval, controls, lifecycle, intake
  │                  pure decision logic, TDD, >80% coverage target
  │
  ├─▶ lib/agents     AgentPort — OpenAI adapter if OPENAI_API_KEY is set,
  │                  else a deterministic keyless mock adapter (default)
  │
  ├─▶ lib/workflow   WorkflowPort-shaped fan-out for multi-domain drafts
  │
  └─▶ lib/data       DataProvider swap point (env-selected)
        │
        └─▶ lib/db   Drizzle ORM — Neon (DATABASE_URL set) or local
                      PGlite (fallback, zero external services)
```

- **Next.js (App Router)** serves both the UI (`app/`) and the API routes
  (`app/api/**`) that mutate state.
- **App-owned ports** (`lib/agents/ports.ts`, `lib/workflow/`) are the only
  seam between application code and AI: `AgentPort` (draft a review, assist
  triage, check completeness) and a deterministic `WorkflowPort`-shaped
  fan-out for running multiple domain drafts at once. Authoritative state
  transitions live in application code and Postgres — never inside an
  adapter.
- **AI SDK adapters**: the real adapter (`lib/agents/openai-adapter.ts`)
  uses the Vercel AI SDK (`generateText` + `Output.object`) against
  `OPENAI_MODEL`; a deterministic, keyless **mock adapter**
  (`lib/agents/mock-adapter.ts`) is the default whenever `OPENAI_API_KEY` is
  unset, so the app and its tests never depend on a live LLM provider.
- **Drizzle ORM over Neon-or-PGlite**: `lib/db/client.ts` uses
  `drizzle-orm/neon-http` when `DATABASE_URL` is set, otherwise a local,
  persistent PGlite instance — same schema, same Drizzle API, zero external
  services required for local dev. See `docs/deploy.md` §3(a) for a known
  gap in Neon-driver transaction semantics.
- **Agents draft, humans decide**: every review, triage suggestion, and
  completeness check produced by an agent is a *draft*. A human reviewer
  edits and signs it; a named, accountable approver — never an agent —
  issues the approval or conditional approval.

## Directory map

| Path | What's there |
|---|---|
| `app/` | Next.js App Router: pages (portfolio board, initiative detail, reviews, controls, audit, admin) and `app/api/**` route handlers for every mutation. |
| `agents/` | Non-executable prompt/schema corpus — the source of truth for each agent capability's system prompt (`instructions.md`) and output shape (`schema.md`), read by `lib/agents` adapters. |
| `lib/domain/` | Shared value types (tiers, roles, lifecycle states, domains) used across the whole codebase. |
| `lib/triage/` | Deterministic risk-tier rules and domain-routing logic (PHI/member-facing/care-influence overlay → tier → required domains). |
| `lib/approval/` | Fast-lane eligibility logic for the deterministic, pre-approved-policy low-risk path. |
| `lib/lifecycle/` | The single source of truth for legal state transitions (role/separation-of-duties/reason checks). |
| `lib/controls/` | Control-evaluation logic (threshold breaches, sustained-window checks). |
| `lib/intake/` | Intake completeness engine (flags missing answers without blocking submission). |
| `lib/agents/` | `AgentPort` type + the real OpenAI adapter and the deterministic mock adapter, selected by `getAgentPort()`. |
| `lib/workflow/` | Deterministic multi-domain draft-review fan-out (`WorkflowPort`-shaped). |
| `lib/services/` | Transactional use cases (initiative lifecycle, monitor run, admin actions) plus the route-guard pipeline (session → rate-limit → budget) that gates every mutating API route. |
| `lib/security/` | Passcode check, session tokens, per-client rate limiting, daily token-budget reservation, input-length caps — all in-memory, pure logic (see `docs/deploy.md` §3(b) for the scaling caveat). |
| `lib/db/` | Drizzle schema + the Neon-or-PGlite client factory. |
| `lib/data/` | The single `DataProvider` swap point (`db` vs `mock`), env-selected. |
| `docs/` | Specs: `plan.md`-adjacent design docs, `demo-script.md`, `deploy.md`, `seed-spec.md`, `intake-spec.md`, `ui-spec.md`, `policies/` (per-domain governance policy corpus). |
| `scripts/` | `seed.ts` — the deterministic seed script implementing `docs/seed-spec.md` (12 initiatives, 9 actors, 16+1 controls, synthetic telemetry). |
| `drizzle/` | Generated SQL migrations, including the DB-level append-only trigger on `audit_events`. |
| `tests/` | Component tests (`tests/ui/`), port compile-tripwire tests, and the required Playwright golden-path spec (`tests/e2e/golden-path.spec.ts`). |

## Tests

```bash
npm test              # Vitest — 438 tests across 35 files
npm run test:e2e      # Playwright — 5 tests (golden-path.spec.ts); boots its own dev server on :3117
npm run typecheck     # tsc --noEmit
npm run lint          # eslint
npx vitest run --coverage   # coverage report (lib/ is the >80%-coverage target per the project's TDD rule)
```

The Playwright suite is the **required golden path**: it walks the
read-only champion storyline (12 seeded initiatives, the Critical-tier
prior-auth summarizer, the synthetic-telemetry label, a 4-row audit query
result, and the 17-control catalog count) end to end against seeded data.

## The hard rules, summarized

The full list lives in `AGENTS.md`; the three that matter most if you only
read one section of this README:

- **Agents draft, they never approve.** Every AI-produced review, triage
  suggestion, or completeness check is a draft a human edits and signs; only
  a named, accountable human approver issues an approval — including the
  deterministic low-risk "fast-lane," which still names an accountable
  approver, never an agent.
- **Public visitors are read-only; every mutation is gated.** There is no
  unauthenticated mutation endpoint. The live/mutable demo workspace
  requires a passcode, an isolated session, an atomic daily budget check,
  and per-client rate limiting — and the keyless mock agent adapter is the
  default, so a shared public URL never incurs LLM cost unless
  `OPENAI_API_KEY` is deliberately set.
- **The audit trail is append-only and evidence-linked, and data is always
  synthetic.** `AuditEvent` rows cannot be updated or deleted at the
  database level (not just in application code), every synthetic telemetry
  panel is labeled as such, and nothing in this repository uses real
  organization names or real PHI/PII.
